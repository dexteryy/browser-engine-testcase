评估 Chrome 对 :has()、:focus-within 等复杂选择器在交互下的支持与性能。

提示：以上选项通过 body:has(#id:checked) 在纯 CSS 中全局生效。

解释说明

测试目标：验证浏览器对复杂关系选择器（如 :has()、:focus-within、:not()、:nth-child）的解析、匹配与在交互状态下的即时更新。

使用方法：

  1. 直接查看静态样式是否正确渲染（如奇数项 hover 后的后缀文本、无徽章项目的下划线风格等）。
  2. 逐个点击卡片右上角的“展开详情”，确认每次点击后内容即时出现/隐藏，无状态不同步。
  3. 使用键盘 Tab 在卡片之间移动焦点，焦点进入某卡片时该容器出现蓝色边框（:focus-within），移出后消失。
  4. 切换上方工具栏选项，观察通过 body:has(#id:checked) 驱动的全局布局变化是否即时生效。

性能评估：打开 DevTools → Performance，录制快速展开/收起过程，关注“Recalculate Style”耗时。如发现明显抖动或卡顿，可在「性能压力」页面进行更强对比。
