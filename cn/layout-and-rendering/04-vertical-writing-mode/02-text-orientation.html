测试用例4-02：字符方向对比

在三个独立竖排列中对比 mixed / upright / sideways

尺寸与排版

数字合字（text-combine-upright）

状态

mixed（默认）

默认 mixed：拉丁字母旋转，东亚字符直立。英文单词 Chrome、CSS、Grid 与数字 2025。

示例：竖排中的半角括号 ( ) 与全角括号（ ）的表现；URL 片段 example.com 仅作文本演示。

日文：縦書きで Web と UI を混在。京都きょうと。

upright

在 upright 下，拉丁字符也直立显示：Chrome CSS3 Grid 1234。

观察大小写字母、长单词的直立可读性与行内对齐。

日文：たてがき + upright。韩文：혼합 upright。

sideways

在 sideways 下，所有字符侧向显示：Chrome CSS Grid 2025。

对比标点旋转、英文单词的连贯性与换行行为。

日文：サイドウェイズ。韩文：사이드웨이즈。

测试目标

在相同内容下对比三种 text-orientation 的差异，观察字符方向、标点定位与换行。

使用方法

  • 调节字体、行高与字间距，观察不同字符集在三种模式下的排版变化。
  • 勾选数字合字（digits 2 / all）以检查竖排数字的合并表现。
  • 对 span.latin 应用强制直立/侧向以验证局部覆盖样式的优先级与渲染。
